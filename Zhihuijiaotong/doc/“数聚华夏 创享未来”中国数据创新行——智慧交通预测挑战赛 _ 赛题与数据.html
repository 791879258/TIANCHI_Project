竞赛题目

移动互联网时代的开启使得每个出行者都成为了交通信息的贡献者，超大规模的位置数据在云端进行处理和融合生成城市全时段，无盲区的交通信息。本届算法挑战赛以“移动互联时代的智慧交通预测”为主题，邀请参赛者基于互联网交通信息建立算法模型，精准预测各关键路段在某个时段的通行时间，实现对交通状态波动起伏的预判，助力社会智慧出行和城市交通智能管控。组委会将通过计算参赛者提交预测值和记录真实值之间的误差确定预测准确率，评估所提交的预测算法。

数据源

移动APP数据实时匿名收集用户地理位置信息， 处理和融合生成城市全时段，无盲区的交通信息。本次大赛将提供城市关键路段（link）的属性信息，路段间网络拓扑结构以及每条路段在历史各时间段内的通行时间，供参赛者建立和测试算法模型。

初赛说明

初赛训练数据

大赛提供132条link的静态信息，以及这些link之间的上下游拓扑结构。同时，大赛提供2016年3月至2016年5月每条link每天的旅行时间，以及2016年6月早上[6:00- 8:00)每条link的平均旅行时间。

初赛第一次比赛预测目标

请选手基于训练数据预测2016年6月份[8:00-9:00)每条link上每两分钟时间片的平均旅行时间。

初赛第二波训练数据

大赛提供132条link的静态信息，以及这些link之间的上下游拓扑结构。同时，大赛提供2017年3月至2017年5月每条link每天的旅行时间，以及2017年6月早上[6:00- 8:00)每条link的平均旅行时间，2min一个时间片。
备注：选手可以选择使用或者不使用第一波数据中的训练数据辅助预测2017年6月早上每条link的旅行时间。

初赛第二次比赛预测目标

请选手基于训练数据预测2017年6月份[8:00-9:00)每条link上每两分钟时间片的平均旅行时间。

复赛说明

复赛训练数据

1） link属性表和link上下游关系表这两部分静态数据保持不变，表名分别为：
        a)  gy_cmp_link_static_info
        b)  gy_cmp_link_in_out_top_info
2） 大赛提供2016年7月，2017年4月至2017年 6月4个月132条link的旅行时间数据, 2min粒度，覆盖全天。
3） 另外,大赛提供2017年7月每天[6:00 – 8:00), [13:00 – 15:00), [16:00 – 18:00)三个时段132条link的旅行时间数据。
备注： （2）和（3）两部分数据存放在ODPS表：gy_cmp_training_traveltime中。

复赛预测任务

复赛第一阶段

预测2017年7月1日至7月15早高峰、日平峰和晚高峰三个时段中各一个小时的数据，预测时段为：
1)  早高峰： [8:00 - 9:00)  
2)  日平峰： [15:00 - 16:00)
3)  晚高峰： [18:00 - 19:00)  
我们会基于选手在这三个时段预测的MAPE综合评估选手能力。
选手需要提交 3 * 15 * 132 * 30 = 178200条数据；预测数据请选手存放在gy_cmp_testing_seg1中。
Table的定义语句为：
CREATE TABLE gy_cmp_testing_seg1
(
link_ID string comment ‘道路ID’,
date_time string comment ‘日期，例如2017-07-01’,
time_interval string comment‘预测时间段，例如：[2017-07-01 08:14:00,2017-07-01 08:16:00)‘，
travel_time double comment ‘预测的旅行时间’
) COMMENT ‘复赛第一阶段预测结果提交表’；

复赛第二阶段

预测2017年7月1日至7月31早高峰、日平峰和晚高峰三个时段中各一个小时的数据，预测时段为：
1)  早高峰：[8:00 - 9:00)
2)  日平峰：[15:00 - 16:00)
3)  晚高峰：[18:00 - 19:00 )
我们会基于选手在这三个时段预测MAPE综合评估选手的能力。
选手需要提交 3 * 31 * 132 * 30 = 368280条数据；预测数据请存放在gy_cmp_testing_seg2中。
CREATE TABLE gy_cmp_testing_seg2
(
link_ID string comment ‘道路ID’,
date_time string comment ‘日期，例如2017-07-01’,
time_interval string comment‘预测时间段，例如：[2017-07-01 08:14:00,2017-07-01 08:16:00)‘，
travel_time double comment ‘预测的旅行时间’
) COMMENT ‘复赛第二阶段预测结果提交表’；
在复赛的第一和第二阶段大赛均会提供一份结果表template gy_cmp_testing_template_seg1和template gy_cmp_testing_template_seg2， 列出所有需要预测的linkID及预测的时间段，但travel_time字段为0，请选手在提交结果前验证自己的结果中是否包含template中列出的所有link_ID和time_interval的组合且唯一，如若预测组合不全，或者不在template表中，成绩无效。

复赛评估方式

1.  从8月18日起，系统基于选手对2017年7月1日至7月15日数据进行评测并排名，系统每天进行一次评测和排名，评测开始时间为当天10:00 AM，按照评测指标从高到低进行排序，每天更新排行榜。
2.  系统在9月12日，系统基于选手对2017年7月1日至7月31日数据进行评测并排名，参赛队伍在访问赛题数据时须注意更换表名，复赛成绩排行榜将选取9月13日起产生的成绩进行重新排名。
3.  复赛截止时间是9月15日10:00AM，复赛成绩排名前5名的选手将安排现场答辩。


评估指标

评估指标计算公式如下：

 

ttp : 参赛者提交的预测通行时间

ttr: 真实通行时间

N: 预测link数量

T_i： 第i个link上的预测时间片数量

MAPE值越低说明模型准确度越高。
